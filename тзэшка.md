# Subscription Service — Техническое Задание v1.0

## Цель проекта

REST-сервис для учета онлайн-подписок пользователей и расчета суммарной стоимости подписок за выбранный период.

Проект выполняется как тестовое задание на позицию Junior Golang Developer.

---

## Основная сущность

### Subscription

```text
id
service_name
price
user_id
start_date
end_date
created_at
updated_at
```

Описание полей:

```text
id            — UUID подписки
service_name  — название сервиса, например "Yandex Plus"
price         — стоимость месячной подписки в рублях
user_id       — UUID пользователя
start_date    — месяц и год начала подписки
end_date      — месяц и год окончания подписки, optional
created_at    — дата создания записи
updated_at    — дата обновления записи
```

## Подсчет суммы подписок

```http
GET /api/v1/subscriptions/total?from=07-2025&to=12-2025
```

С фильтрами:

```http
GET /api/v1/subscriptions/total?from=07-2025&to=12-2025&user_id=...&service_name=Yandex Plus
```

Response:

```json
{
  "total": 2400
}
```

## Бизнес-правила

```text
service_name обязателен
price должен быть > 0
user_id должен быть валидным UUID
start_date обязателен
end_date optional
end_date не может быть раньше start_date
формат дат в API: MM-YYYY
```

В базе даты храним как DATE, где день всегда первое число месяца.

Пример:

```text
07-2025 → 2025-07-01
```

## PostgreSQL схема

Таблица:

```text
subscriptions
```

Поля:

```text
id UUID PRIMARY KEY DEFAULT gen_random_uuid()
service_name TEXT NOT NULL
price INTEGER NOT NULL CHECK (price > 0)
user_id UUID NOT NULL
start_date DATE NOT NULL
end_date DATE NULL
created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
```

Индексы:

```text
idx_subscriptions_user_id
idx_subscriptions_service_name
idx_subscriptions_start_date
idx_subscriptions_end_date
```
