> Test Task for Junior Golang Developer (Effective Mobile)

# 1. Цель проекта

Разработать REST-сервис для учёта онлайн-подписок пользователей.

Сервис должен:

* хранить информацию о подписках пользователей;
* поддерживать полный CRUDL набор операций;
* рассчитывать суммарную стоимость подписок за произвольный период;
* предоставлять Swagger документацию;
* запускаться через Docker Compose.

---

# 2. Границы ответственности сервиса

Сервис отвечает только за подписки.

Сервис НЕ отвечает за:

* регистрацию пользователей;
* авторизацию;
* проверку существования пользователя;
* хранение профилей пользователей;
* интеграции с внешними сервисами.

Поле `user_id` рассматривается как внешний UUID, полученный из другого сервиса.

# 4. Архитектурные принципы

Проект строится по простой слоистой архитектуре.

```text
HTTP Request
    ↓
Middleware
    ↓
Handler
    ↓
Service
    ↓
Repository Interface
    ↓
PostgreSQL Repository
    ↓
PostgreSQL
```

---

## Handler

Отвечает за:

```text
HTTP
JSON
query parameters
path parameters
status codes
```

Не содержит бизнес-логики.

---

## Service

Отвечает за:

```text
валидацию
бизнес-правила
парсинг дат
расчёт стоимости
```

Не знает SQL.

---

## Repository

Отвечает за:

```text
SQL
PostgreSQL
сканирование строк
работу с транзакциями
```

Не содержит бизнес-логики.

---

## Domain

Содержит:

```text
бизнес-сущности
доменные ошибки
```

Не зависит от HTTP и PostgreSQL.

# 7. Работа с датами

Требование ТЗ:

```text
MM-YYYY
```

Пример:

```text
07-2025
```

---

В API:

```json
{
  "start_date": "07-2025"
}
```

---

В PostgreSQL:

```text
2025-07-01
```

Используется первое число месяца.

Причины:

```text
проще фильтровать
проще сравнивать
проще строить SQL-запросы
```

# 8. Схема базы данных

## Таблица subscriptions

```sql
CREATE TABLE subscriptions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

    service_name TEXT NOT NULL,

    price INTEGER NOT NULL
        CHECK (price > 0),

    user_id UUID NOT NULL,

    start_date DATE NOT NULL,

    end_date DATE NULL,

    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),

    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),

    CONSTRAINT chk_dates
        CHECK (
            end_date IS NULL
            OR end_date >= start_date
        )
);
```

---

## Индексы

```sql
CREATE INDEX idx_subscriptions_user_id
ON subscriptions(user_id);

CREATE INDEX idx_subscriptions_service_name
ON subscriptions(service_name);

CREATE INDEX idx_subscriptions_user_start_date
ON subscriptions(user_id, start_date);
```

# 9. Архитектурные решения

## UUID

Используется для всех публичных идентификаторов.

Причина:

```text
подходит для распределённых систем;
не раскрывает количество записей;
соответствует ТЗ.
```

---

## INTEGER для цены

```text
price INTEGER
```

Причина:

```text
ТЗ запрещает копейки.

FLOAT использовать нельзя.
```

---

## DATE вместо строки

В API:

```text
MM-YYYY
```

В БД:

```text
DATE
```

Причина:

```text
SQL работает с датами лучше, чем со строками.
```

# 11. Бизнес-логика расчёта

Подписка участвует в расчёте если её период пересекается с:

```text
[from, to]
```

Условие:

```text
start_date <= to

AND

(end_date IS NULL OR end_date >= from)
```

---

Стоимость:

```text
price
*
количество месяцев пересечения
```

---

Итог:

```text
SUM(price * overlap_months)
```

---

# 12. Валидация

Проверяется:

```text
service_name обязателен

price > 0

user_id валидный UUID

start_date обязателен

формат даты MM-YYYY

end_date >= start_date

from <= to

limit <= 100
```
